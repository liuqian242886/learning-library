L0

课程介绍 · Introduction

讲师向你介绍本课程要学什么，以及 Skills 在整个 AI 生态中扮演什么角色。

课程核心问题

随着 AI 应用的普及，一个问题越来越突出：如何让 AI 的专业能力变得可复用、可组合、可分享？

Skills（技能）就是 Anthropic 给出的答案——一种让 AI Agent 拥有"特定领域知识包"的机制。

本课程覆盖内容
  • 在 Claude AI / Desktop 中创建和使用技能
  • 通过 Claude Messages API 以编程方式使用技能
  • 在 Claude Code 命令行工具中使用技能
  • 通过 Claude Agent SDK 构建智能代理应用
什么是 Skills？

Skill（技能）= 一个包含 SKILL.md 文件的文件夹。这个文件夹可以包含：

指令说明（Markdown） 可执行脚本（Python等） 模板资产文件 图片/Logo等资源 引用文档
技能的核心价值
  • 可移植：在 Claude AI、Claude Code、API、SDK 中格式完全相同
  • 可组合：多个技能可以组合使用，构建复杂工作流
  • 渐进式：按需加载，不浪费上下文窗口
  • 开放标准：不只是 Anthropic 产品，Gemini CLI、Codex 也支持

为什么使用 Skills（上）· Why Use Skills Part I

本节通过一个完整的营销活动数据分析演示，让你看到技能是如何从零开始创建并投入使用的。

💡 这一节是整门课最重要的"快速入门"演示。讲师用一个真实业务场景，完整走了一遍"发现需求 → 创建技能 → 使用技能"的全流程。
核心演示场景：营销活动分析

讲师的场景是：每周需要分析一个营销活动的 Excel 数据，评估每个渠道（搜索、社交、展示广告、邮件）的表现，并给出预算调整建议。

第一步：用自然语言描述需求，让 Claude 生成 SKILL.md

讲师直接告诉 Claude：

"我每周需要分析营销数据。把我用来分析的提示词以及预算重新分配的规则，打包成一个可复用的技能。"

Claude 使用内置的 skill-creator 技能 来帮助创建 SKILL.md。创建出来的文件包含：

  • YAML Frontmatter：技能名称 analyzing-marketing-campaign + 描述
  • 数据分析步骤：如何读取 CSV，分析各渠道指标
  • 预算重分配规则：引用了一个外部规则文档
  • 输出格式：要求生成 Excel 格式的分析报告
第二步：将技能上传到 Claude Settings
1
在 Claude.ai 或 Claude Desktop 中，进入 Settings → Capabilities（能力）
2
在 Skills 区域点击 上传技能文件夹（包含 SKILL.md 的目录）
3
技能上传后，每次对话 Claude 都会自动知道该技能的存在
4
当用户说"帮我分析这周的营销数据"，Claude 自动触发技能
第三步：技能实际运行效果

没有技能时：

  • 每次都要重新描述分析方法
  • 输出格式不一致
  • 忘记某些分析步骤
  • 无法在团队间共享

有技能后：

  • 只需上传 CSV 文件
  • Claude 自动调用 Excel 技能生成报告
  • 输出格式每次一致且专业
  • 技能可复制给所有同事使用

为什么使用 Skills（下）· Why Use Skills Part II

本节从理论层面深入讲解 Skills 的设计理念：开放标准、渐进式披露、可组合工作流。

Skills 是开放标准（视频原文要点）
"Similar to the model context protocol, skills themselves are an open standard that can be used across many different AI applications."

Skills 最初由 Anthropic 创建，现已成为跨平台的开放标准，支持：

Claude Code Gemini CLI Codex Open Code 更多 Agent 产品
为什么历史上的 Agent 方式不够好？

过去构建 Agent 的方式是：为每个领域创建单一用途的专用 Agent（编码 Agent、研究 Agent、财务 Agent……）

问题：

  • 底层其实只需要简单的脚手架（bash + 文件系统）
  • 缺少的是上下文和领域专业知识
  • 维护大量专用 Agent 成本高

解决方案：简单 Agent + Skills，技能提供领域知识和工作流程。

Skills 能解决三类问题
①
域名专业知识：告诉 Claude 你的公司、团队要怎么做某件事（例如：按照公司格式写报告）
②
可重复工作流：在非确定性系统中，通过精确的步骤说明，让输出更可预测
③
新能力：让 Agent 能做它原本不知道怎么做的事，如生成 PPT、填写 PDF 表单
🔑 渐进式披露（Progressive Disclosure）——本课最重要概念之一

问题：如果你有 100 个技能，不能全部塞进上下文窗口。

讲师把上下文窗口比作公共资源：

"We like to think of the context window as a public good. The more data that we add to the context window, the more tokens we consume, the faster our context window fills up, and the likelihood of context degradation or incorrect responses potentially increases."

渐进式披露的三阶段加载：

阶段 1
技能名称 + 描述
始终在上下文
→ 触发技能 →
阶段 2
加载完整 SKILL.md
按需加载
→ 需要时 →
阶段 3
脚本/文档/资产
仅在需要时执行

Skills 与 Tools / MCP / Subagents 的关系

AI 生态中有很多概念容易混淆。本节课用清晰的类比和实际案例，彻底讲清楚它们的区别和协作关系。

Skills vs MCP（模型上下文协议）
  • MCP：负责连接外部数据系统，把外部工具和资源引入 Agent（数据库、Google Drive、BigQuery 等）
  • Skills：负责告诉 Agent 如何使用那些工具和数据，定义可重复的工作流
"Think of the model context protocol as bringing in all of the underlying tooling that we need, and the skill as the set of instructions to put those tools together to build particular workflows."
Skills vs Tools（工具）

工具类比：锤子、锯子（底层执行能力）

技能类比：如何建一个书架（使用工具的方法论）

  • Tools（工具）：底层访问系统的方式，始终在上下文窗口中
  • Skills（技能）：使用工具的高级指令，按需渐进式加载
  • Skills 内部可以引用需要执行的脚本，脚本调用底层工具能力
Skills vs Subagents（子代理）
  • Subagents：主 Agent 派生出来的子 Agent，拥有独立的上下文窗口，可以并行执行任务
  • Skills：提供给 Agent 的知识和工作流程，主 Agent 和子 Agent 都可以使用技能

最佳实践：

"Use skills for procedural, predictable workflows and subagents for full agentic logic only when necessary for specialized tasks."
📌 综合案例：客户洞察分析器（视频原文演示）

课程用这个案例完整展示四者如何协作：

🔌 MCP
连接客户数据库，提供原始数据工具
📋 Skills
定义如何分类反馈、总结发现、分析访谈的方法
👥 Subagents
并行分析多个客户访谈/调查，加速处理
🔧 Tools
执行具体操作：读文件、写文件、执行 bash

探索内置技能库 · Exploring Pre-Built Skills

Anthropic 已经在 GitHub 上开源了一批生产级技能。本节带你深入看这些技能的结构，并演示如何把它们组合起来完成复杂工作流。

官方技能仓库：github.com/anthropics/skills

仓库分为两大类技能：

📄 文档技能（Document Skills）
这些技能内置在 Claude AI 中，无法关闭
Excel (xlsx) PowerPoint (pptx) Word (docx) PDF
🔧 示例技能（Example Skills）
默认关闭，可在 Claude 设置中开启。skill-creator 默认开启
skill-creator ✓ 其他示例（默认关）
PowerPoint 技能详解（视频原文）

讲师在 GitHub 上打开 pptx 技能，展示其结构：

  • YAML Frontmatter：name + description
  • Overview：用户可能需要创建/编辑/分析 PPT 文件
  • 颜色和字体：设计规范和调色板
  • 设计原则：当用户不指定时 Claude 从预设中选取
  • 脚本引用：只在需要时加载和执行
"Remember, these are not executed right out of the box. These are only loaded and executed when necessary."
skill-creator 技能详解

一个用来创建其他技能的"元技能"。包含三个 Python 脚本：

  • init_skill.py：从模板创建技能，填充 YAML Frontmatter 和占位符
  • package_skill.py：将技能打包为 zip 文件
  • validate_skill.py：验证 SKILL.md 存在、YAML 格式正确
"It's important to make sure we still need to give the context necessary when using skill-creator. It's extremely effective but we still need to be intentional about the prompt and data that goes in."
完整演示：BigQuery 数据 + 品牌指南 + PowerPoint 三技能工作流

视频中讲师演示了一个完整的多技能组合工作流：

1
修改营销分析技能：用 skill-creator 把原本读取 CSV 的技能改为从 BigQuery 读取数据，技能中指定 MCP 服务器名称和工具名
2
创建品牌指南技能：上传品牌手册（色彩、字体、Logo），让 skill-creator 生成包含品牌资产的新技能
3
三技能协作：用"营销分析技能"从 BigQuery 查数据 → 用"品牌指南技能"获取样式 → 用内置"PowerPoint 技能"生成演示文稿
4
最终效果：完整品牌风格的数据驱动 PPT，包含效果分析、漏斗分析、执行摘要，可直接上传 Google Drive 分享

创建自定义技能 · Creating Custom Skills

本节是整门课最"干货"的一节——深入讲解 SKILL.md 的结构规范、最佳实践，并演示两个自定义技能的完整创建和评估流程。

SKILL.md 文件结构规范
YAML Frontmatter
必填：name（技能名称）、description（技能描述）
可选：license、compatibility、自定义键值对
Markdown 正文
技能主要内容，包含分步指令、边缘情况处理、工作流程
scripts/
需要执行的代码（Python等），含错误处理和说明文档
references/
附加文档或参考文件，内容较长时建议在技能中指明要全部读取
assets/
模板、图片、Logo、数据文件、Schema 等
Name 和 Description 最佳实践
  • Name：只能含小写字母、数字和连字符；建议用 动词-ing 格式，如 analyzing-marketing-campaign
  • Description：描述技能做什么，以及何时使用它；包含关键触发词
  • Name 和 Description 是 Claude 决定是否触发技能的依据，质量至关重要
"Your name and description are not only how Claude can analyze what your skill does, but also detect when to use that particular skill."
Markdown 正文最佳实践
  • 写分步指令，比笼统说明更有效
  • 明确说明边缘情况，和跳过某步骤的条件
  • 建议控制在 500 行以内，更多内容引用外部文件
  • 路径使用正斜杠（/），确保跨平台兼容
  • 考虑自由度：遵守规范→低自由度；创意输出→高自由度
两个自定义技能案例（视频原文演示）

📝 generating-practice-questions 技能

  • 用途：从讲义（PDF/LaTeX/Markdown）自动生成练习题
  • 支持输入格式：PDF、LaTeX、Markdown
  • 题型顺序：判断题 → 说明题 → 编程题 → 应用题
  • 输出：引用 assets/ 中的模板文件（markdown_template.md / latex_template.md）
  • skill-creator 评分：9/10，建议：可适当精简

📊 analyzing-time-series 技能

  • 用途：分析时序 CSV 数据特征，为预测做准备
  • 包含三个 Python 脚本：visualize.py、autocorr.py、diagnose.py
  • diagnose.py 按序执行：数据质量 → 分布 → 平稳性 → 季节性 → 趋势 → 自相关 → 转换建议
  • 输出：精确的文件树结构（summary.txt + 图表）
  • skill-creator 评分：高分，Frontmatter 质量、简洁性表现优秀
如何对技能进行单元测试？

视频展示了一个测试框架思路：

  1. 定义多个测试查询（例如：生成 Markdown 格式/LaTeX 格式的题目）
  2. 传入符合规范的输入文件
  3. 检查期望行为：是否使用正确库、提取了学习目标、生成了对应题型
  4. 验证输出文件格式是否符合 assets/ 中的模板
  5. 收集人工反馈，在多模型下测试

Skills + Claude Messages API · 以编程方式使用技能

本节展示如何在 Jupyter Notebook 中，通过 Claude API 以代码方式运行技能，并配合 Code Execution Tool 和 Files API 完成完整工作流。

关键前提：Claude AI vs Claude API 的区别
  • Claude AI/Desktop：自动提供代码执行容器和文件系统，技能开箱即用
  • Claude API：需要手动启用 Code Execution Tool 才能使用技能
  • 两个环境中技能的格式完全相同，只是启用方式不同
"Skills that you create in Claude AI and Claude Desktop are not shared in the Claude API or Claude Code."
Code Execution Tool（代码执行工具）

给 Claude 一个安全隔离的容器来执行代码和操作文件系统。

  • 支持：Bash/Shell 命令、创建/查看/编辑文件
  • 限制：无互联网连接（API 中）、固定 RAM/磁盘/CPU 上限
  • 预装了常用 Python 库
  • Claude AI/Desktop 中可访问互联网，API 中不行

API 调用关键代码结构（视频原文演示）
# 1. 必需的 beta 请求头 betas = ["skills-2025-04-01", "code-execution-20250522-2", "files-api-2025-04-14"] # 2. 上传技能目录，获取 skill_id skill = client.beta.skills.create(files=files_from_dir("./my-skill")) skill_id = skill.id # 3. 调用 API，传入技能和工具 response = client.beta.messages.create( model="claude-sonnet-4-5", betas=betas, container={"skills": [{"id": skill_id, "version": "latest"}]}, tools=[{"type": "code_execution_20250522"}], messages=[{"role": "user", "content": [ {"type": "text", "text": "Generate practice questions"}, {"type": "document", "source": {"file_id": file_id}} ]}] )

  • 最终：包含统计分析、可视化图表的专业 Word 文档

Skills + Claude Code · 命令行环境中使用技能

Claude Code 是 Anthropic 的命令行 AI 编码工具。本节展示如何在 Claude Code 环境中安装、管理和使用技能。

💡 上一节（L5）在演示 skill-creator 评估时已短暂展示过 Claude Code。本节是完整的 Claude Code + Skills 专题。
Claude Code 中技能的存放位置

与 Claude AI（Settings 上传）不同，Claude Code 使用文件系统目录管理技能：

# 项目级技能（只对当前项目生效） .claude/skills/my-skill/SKILL.md # 全局技能（对所有项目生效） ~/.claude/skills/my-skill/SKILL.md

在 .claude/settings.json 中管理已启用的插件：

{ "enabledPlugins": ["anthropics/skills/skill-creator"] }
从 Marketplace 安装技能

Claude Code 支持通过 Marketplace 安装技能插件：

1
在 Claude Code 中添加 Marketplace：anthropics/skills
2
Marketplace 包含两个集合：document-skills（Excel/PPT/Word/PDF）和 example-skills（skill-creator 等）
3
选择 Project Scope 安装（只影响当前项目）
4
重启 Claude Code，settings.json 中自动添加 enabledPlugins
5
使用 /skills 命令查看所有已安装技能
与 Claude AI 的关键差异
  • Claude Code 没有内置文档技能（Excel/PPT 等），需要手动从 Marketplace 安装
  • Claude Code 自带完整文件系统访问权限，无需额外配置代码执行环境
  • 技能格式完全相同，可以从 Claude AI 直接迁移到 Claude Code

结果：两个子代理并行运行，分别评估两个技能：

  • generating-practice-questions：9/10 分
  • analyzing-time-series：高分，Frontmatter 质量和简洁性表现优秀

Skills + Claude Agent SDK · 构建多代理研究系统

本节是整门课最"高级"的一节。从零构建一个通用研究代理，使用 Agent SDK 协调多个子代理并行工作，并通过 MCP 将结果写入 Notion。

什么是 Claude Agent SDK？
"The Claude Agent SDK is a programmatic way of building your own agentic applications that use the same internal harness that Claude Code does."

简单说：Agent SDK 让你用 Python 代码构建出和 Claude Code 内核一样的 Agent 应用，可以完全控制代理的行为、工具权限和子代理结构。

系统架构：通用研究代理

本节构建的系统由 1 个主代理 + 3 个子代理 组成：

🎯 主代理（Orchestrator）：接收用户请求，协调三个子代理，综合所有结果
📚 docs_researcher：使用 WebSearch + WebFetch 查找和抓取官方文档
🔍 repo_analyzer：使用 WebSearch + Bash（git clone）分析 GitHub 仓库结构
🌐 web_researcher：搜索教程、YouTube 视频、社区资源
learning-a-tool 技能的作用

本节使用的技能叫 learning-a-tool，它的特殊之处在于：只作用于主代理，不给子代理使用。

技能内容包含：

  • 研究阶段工作流：每个子代理需要收集哪些信息（精确指定）
  • 渐进式学习结构：从 Overview → 安装 → 核心概念 → 实践模式 → 进阶
  • 输出格式规范：README、resources、学习路径、代码示例的目录结构
"We're using this skill as a way to create a predictable pattern so that the main agent knows the ideal workflow and what and how to dispatch subagents."

GLOSSARY
术语表
课程中涉及的核心术语解释，均基于视频原文内容。

Skill（技能）

一个包含 SKILL.md 的文件夹，可以含有指令、脚本、模板、图片等。让 AI Agent 拥有可复用的专业知识和工作流程。

SKILL.md

每个 Skill 文件夹中必须有的文件。顶部是 YAML Frontmatter（name + description），下面是 Markdown 格式的指令内容。

Progressive Disclosure（渐进式披露）

技能的三阶段加载机制：先只加载名称/描述 → 触发后加载 SKILL.md → 按需加载脚本和资产文件，以节省上下文窗口。

YAML Frontmatter

SKILL.md 顶部用三条横线包围的 YAML 区块，包含必填的 name、description 字段，以及可选的 license、compatibility 等。

skill-creator

Anthropic 提供的一个内置技能，专门用来帮你创建其他技能。包含最佳实践规范，可在 Claude AI、Claude Code 中使用。

Code Execution Tool

Claude API 中的工具类型，给 Claude 提供一个沙盒容器来执行代码和操作文件系统。使用技能时必须启用。

Files API

Claude API 提供的文件管理接口，用来上传输入文件并下载技能生成的输出文件，通过 file_id 引用。

MCP（模型上下文协议）

Model Context Protocol。连接 AI Agent 与外部数据系统（数据库、Google Drive、Notion 等）的开放协议，与技能互补。

Subagent（子代理）

主代理派生的独立代理，拥有自己的上下文窗口和工具权限，可以并行执行任务并将结果汇报给主代理。

AgentDefinition

Claude Agent SDK 中定义子代理的类，包含描述（description）、系统提示（prompt）、允许使用的工具（tools）。

setting_sources

Claude Agent SDK 中指定技能搜索路径的参数，可设为 "user"（用户目录）和/或 "project"（项目目录）。

Container（容器）

Claude API 调用中传递技能的参数位置，格式为 container={"skills": [{"id": skill_id}]}，告诉 API 使用哪些技能。
