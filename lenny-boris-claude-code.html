ORIGIN STORY
🌱 Claude Code 的诞生：从一个周末原型开始
起点 · 2025年初
周末黑客实验
Boris 用一个周末做了一个简单的终端原型——让 Claude 直接在命令行里读写文件、运行代码。没有精心设计，只是一个快速验证想法的 hack。
内部实验
Anthropic 工程师开始自己用
原型在 Anthropic 内部传开，越来越多工程师把它集成进日常工作流。这种未预期的自发采用让 Boris 意识到这个工具触及了真实需求——这正是他所说的"潜在需求"信号。
短暂离开 · 2025年中
Boris 加入 Cursor，两周后返回
Boris 曾短暂加入 Cursor，但仅两周后便回到 Anthropic。他认为，Anthropic 在"模型+工具"的垂直整合上有独特优势——Claude Code 可以直接访问最强的模型能力，这是外部工具难以复制的。
高速增长 · 2026年初
占 GitHub 公开提交的 4%
截至播出时，Claude Code 生成的代码已占全球 GitHub 公开提交的 4%，且日活用户环比翻倍。Spotify 报告其最优秀的开发者自 2025年12月起已完全不再手写代码。

🎵 Spotify 案例

Spotify 公开表示，其最优秀的工程师自 2025 年 12 月起已完全不再手写代码。这不是替代，而是进化——他们把时间用在架构决策、产品方向和系统设计上。

⚡ Boris 个人工作流

Boris 本人自 2025 年 11 月起就没有再手动编辑过任何一行代码。他每天通过 Claude Code 生成并合并 10 到 30 个 Pull Request，工作重心完全转向了产品方向和决策。

🏢 全团队都在写代码

在 Claude Code 团队里，不只是工程师在用 Claude Code——产品经理、设计师、财务、数据科学家都在借助它构建功能和工具。这印证了 Boris 的"构建者"理念。

🚀 Cowork：10 天从零上线

Cowork（面向非开发者的桌面 AI 产品）从构思到上线只用了 10 天，全程由 Claude Code 构建。这个速度在传统开发模式下需要数月。

CORE THESIS
🧩 "编程已被解决"——这到底意味着什么？
"Coding is largely solved. The primary challenge now is prioritizing ideas for development."
— Boris Cherny，播客节目中

Boris 并不是说"不再需要人类"，而是说：把想法转化为可工作代码这件事，已经不再是瓶颈。瓶颈从"怎么写代码"转移到了"构建什么"和"为什么构建它"。

✅ Boris 的核心论点
将想法转化为代码的能力不再是瓶颈
工程师的身份正在从"打字员"进化为"构建者"
判断力、优先级和产品直觉的价值大幅提升
2026 年底"软件工程师"职称可能开始消失
每个人都可以成为构建者（PM、设计师、运营均如此）
⚠️ 争议与反驳
复杂系统架构设计仍需深厚技术背景
调试 AI 生成代码需要理解代码逻辑
"打字"只是编码的一小部分，系统思维才是核心
某些批评者认为 Boris 混淆了"编码"与"打字"
代码审查和安全仍需专业工程师把关
📖 印刷机类比
Boris 用古腾堡印刷机来类比 AI 对编程的影响：印刷机没有消灭作家，而是让更多人能够写作和出版。AI 编码工具同理——不是取代工程师，而是让更多人能参与构建软件。
PRODUCT PRINCIPLES
🎯 Boris 的反直觉产品原则

Boris 分享了他在 Claude Code 团队践行的核心原则——其中很多与传统产品智慧相悖，但实践证明极为有效。

01
🪙 故意"欠资"团队
给团队充足的计算资源（无限 token），但在人员和时间上故意保持精简。限制反而激发创造力，避免"人多乱"的熵增。
02
🔑 给工程师无限 Token
让工程师在无 token 限制的环境下使用 AI——这是发现真实产品边界和下一步方向最快的方法，也能建立对产品的真实信任。
03
🔭 追踪"潜在需求"
当用户用一个工具做它"不应该"做的事时，这就是金矿。Boris 定义潜在需求为"用户用产品做非设计用途的行为"——它指向了下一个产品机会。
04
🧪 实验优先
AI 产品的变化速度远超任何规划周期。最重要的能力是快速实验、快速学习。不要在尚未验证的方向上过度投入。
05
🏗️ 自己用自己的产品
Claude Code 是用 Claude Code 构建的，Cowork 也是。这不只是营销说法——它确保团队能第一手感受用户痛点，让反馈循环极短。
06
🌍 拥抱"三层 AI 安全"
Anthropic 的 AI 安全体系分三层：模型层（训练期价值观）、产品层（使用政策）和社会层（外部规范与法规）。三者缺一不可。
💡 新成员必知三原则
Boris 对每位新加入团队的成员分享三条原则：① 用产品本身构建产品；② 潜在需求是最重要的信号；③ 实验比规划更重要。这三条也是 Claude Code 快速成功的核心密码。
FUTURE OF WORK
🔮 AI 时代：哪些职业正在被重塑？

在编程之后，Boris 认为 AI 将依次重塑以下职业领域——顺序大致是"工具化程度"从高到低：

1
软件工程师（已进行中）——"打字"部分已被 AI 接管，未来角色是架构师和构建者。预计"软件工程师"职称到 2026 年底开始淡出。
2
知识工作者·文档/分析——法律文件、财务分析、报告写作等重复性知识工作将快速自动化。Cowork 正是针对这一场景设计的。
3
产品经理 & 设计师——AI 工具使 PM 和设计师能直接构建原型甚至产品，大幅缩短从构想到落地的周期。
4
招聘与人力资源——Metaview 等 AI 招聘平台正在重构面试、筛选和候选人沟通流程（节目赞助商之一）。
5
影响力·人际关系——Jessica Fain（下一期嘉宾）认为"影响力"是 AI 难以替代的最后一种核心技能，面向人的沟通和信任建立仍是人类优势。
📊 调查数据
节目中引用了 Lenny 在 X（推特）上发起的民调：哪些职业的从业者在 AI 时代反而更享受工作？ 结果显示：使用 AI 工具最深入的开发者群体，工作满意度反而显著提升——因为他们从重复性任务中解放了出来。

00:00 Introduction to Boris and Claude Code ▼

Lenny 介绍 Boris Cherny 是 Claude Code 的创始人和负责人。Boris 回顾了自己在 Anthropic 的角色——他最初是一名工程师，Claude Code 源于他对"如何让 AI 真正帮助开发者"这一问题的实验性探索。

背景
Boris 是 TypeScript 领域知名工程师（《Programming TypeScript》作者），在加入 Anthropic 之前曾在 Facebook 工作多年。
03:45 Why Boris briefly left Anthropic for Cursor（and what brought him back） ▼

Boris 坦诚地聊了他曾短暂加入 Cursor 的经历。Cursor 是当时市场上最热门的 AI 代码编辑器，估值超过 30 亿美元。然而仅两周后他就决定回到 Anthropic。

关键洞察
Boris 认为，在"模型提供者"内部构建开发工具有独特优势：可以直接访问最新模型能力、更快速地迭代提示和微调。这种垂直整合的优势是外部工具难以复制的护城河。

一年时间，Claude Code 从一个终端原型成长为占全球 GitHub 公开提交 4% 的工具，日活用户上月翻倍。Boris 回顾了这一年最出乎意料的发展：最大的用户群体并非他们最初预想的独立开发者，而是大型企业的工程团队。

08:41 The origin story of Claude Code ▼

Claude Code 的诞生是一个经典的"内部工具外部化"故事。Boris 做了一个可以让 Claude 读写本地文件、运行 shell 命令的原型，只是为了自己用。当这个工具在 Anthropic 内部广泛传播后，他意识到这是一个真实的产品机会。

潜在需求信号
非预期的内部采用，是最强的产品-市场契合信号。用户用你的工具做它"不应该"做的事，意味着他们正在为一个未满足的需求找解决方案。
13:29 How fast AI is transforming software development ▼

AI 对软件开发的改变速度远超所有预期。Boris 引用 Spotify 案例：其最优秀的开发者自 2025年12月起已完全停止手写代码，但生产力并未下降，反而显著提升。他认为"AI 编程"在精英工程师群体中的渗透率已经相当高。

Spotify 数据点
Spotify 最优秀的开发者在 2025年12月后完全使用 AI 编写代码（TechCrunch 报道），是 AI 重塑软件开发的标志性里程碑。
15:01 The importance of experimentation in AI innovation ▼

Boris 强调：在 AI 时代，能力的进化速度远超任何产品规划周期。因此，团队的核心竞争力不是"更好的计划"，而是"更快的实验速度"。Claude Code 的很多关键功能，都是从意料之外的用户行为中发现的。

16:17 Boris's current coding workflow（100% AI-written） ▼

Boris 详细描述了他的日常工作流程：他通过自然语言向 Claude Code 描述需要的功能，Claude Code 生成整个 PR，他审查后合并。他已经超过三个月没有手动输入过任何一行代码。

每日 PR 数量
Boris 每天合并 10 到 30 个 Pull Request，全部由 Claude Code 生成。他的主要工作变成了：定义需求、审查代码逻辑、判断产品方向。
17:32 The next frontier ▼

编程之后，下一个前沿是什么？Boris 认为是"所有知识工作的自动化"——不只是写代码，而是所有可以被拆解为清晰步骤的脑力劳动。Cowork 就是这个愿景下的产物，专门为非技术用户在文件、数据和任务管理层面提供 AI 辅助。

22:24 The downside of rapid innovation ▼

快速创新也有代价：技术负债积累更快、团队适应压力更大、用户可能跟不上产品迭代节奏。Boris 坦诚地聊了 Claude Code 团队在快速扩张过程中遇到的挑战，包括文档滞后、工具稳定性问题等。

核心张力
快速实验 vs. 产品稳定性——Claude Code 团队的解法是：用 Claude Code 本身来维护和修复 Claude Code 的 bug，形成自我改进的闭环。
24:02 Principles for the Claude Code team ▼

Boris 分享了他向每一位新加入团队成员传递的三条原则：

1
用产品本身构建产品——这不仅能发现 bug，更能建立团队对产品的真实信任和深度理解。
2
潜在需求是最重要的产品信号——用户用工具做非预期的事情，比任何用户访谈都更有价值。
3
实验比规划更重要——在 AI 时代，世界变化太快，最好的策略是保持小步快跑，快速学习。
26:48 Why you should give engineers unlimited tokens ▼

这是本期最反直觉的建议之一。Boris 认为，给工程师无限 token 配额（不限制 AI 使用量）是最好的产品投资——它让工程师能够在没有顾虑的情况下全力实验，同时也是发现产品边界和下一步方向最快的方法。

"The token cost is trivial compared to the engineering time cost. Let your engineers use AI without limits."
— Boris Cherny
27:55 Will coding skills still matter in the future? ▼

Boris 的回答：会，但方式不同。 你不再需要记忆语法，但你需要理解系统设计原则、能够判断 AI 生成代码的质量、能够描述你想要什么。代码素养（code literacy）变得比代码编写（code writing）更重要。

新型技能栈
未来工程师需要的核心能力：系统思维、需求拆解能力、AI 输出判断力、跨领域沟通能力——以及知道什么时候让 AI 做、什么时候自己做。
32:15 The printing press analogy for AI's impact ▼

Boris 用古腾堡印刷机类比 AI 对编程的影响：印刷机出现之前，写作是精英技能，只有少数人掌握。印刷机让写作能力大众化，但也创造了出版业、新闻业等全新行业。AI 编码工具同理——它不会消灭工程师，而是让更多人能够构建软件，并催生全新的职业形态。

36:01 Which roles will AI transform next? ▼

在软件工程之后，Boris 认为 AI 将依次重塑：知识工作者（文档、分析）→ 产品/设计 → 招聘/人力 → 客户服务 → 教育。他特别指出，需要深度人际信任和情感连接的工作，是 AI 最难替代的领域。

40:41 Tips for succeeding in the AI era ▼

Boris 给出了在 AI 时代脱颖而出的实用建议：

1
立即开始深度使用 AI 工具，不要只是浅尝辄止
2
专注发展"判断力"而非"执行力"——什么值得做比怎么做更重要
3
跨领域学习，成为"T型人才"——AI 已经帮你覆盖了深度，宽度是你的竞争力
4
主动把自己放在 AI 转型最前沿的团队和公司
44:37 Poll: Which roles are enjoying their jobs more with AI ▼

Lenny 在 X 上发起的调查显示，深度使用 AI 工具的工程师群体，工作满意度反而显著提升。这与"AI 会让工作更枯燥"的直觉相反——原因是 AI 承担了重复劳动，让人得以专注于更有创造性的工作。

46:32 The principle of latent demand in product development ▼

这是本期最有价值的产品洞察之一。Boris 定义"潜在需求"为：用户用产品做其非设计用途的方式，揭示了他们真正想要什么。

Claude Code 的潜在需求案例
早期用户开始用 Claude Code 管理文件、写邮件、整理文档——这些都不是设计用途。这个信号直接催生了 Cowork 的立项：为非开发者提供 AI 文件和任务管理工具。

另一个案例：用户开始在 Claude Code 里管理 GitHub Issue、发 PR 评论——这直接推动了 GitHub 集成的优先级提升。

51:53 How Cowork was built in just 10 days ▼

Cowork（Anthropic 面向非开发者的桌面 AI 工具）从立项到上线只用了 10 天，全程由 Claude Code 构建。

Cowork 定位
Cowork 是 Claude Code 面向非技术用户的"降维"产品——不需要懂代码，只需要描述任务，AI 就能完成文件整理、数据处理、邮件草稿等工作流自动化。它的诞生直接来源于 Claude Code 用户的"潜在需求"信号。

54:04 The three layers of AI safety at Anthropic ▼

Boris 解释了 Anthropic 的三层 AI 安全体系：

1
模型层（训练期）——在模型训练阶段植入核心价值观和安全边界，这是最根本的防线。
2
产品层（使用政策）——通过使用条款、API 政策和产品设计约束，防止滥用。
3
社会层（外部规范）——与监管机构、研究者和社会各方协作，建立外部规范和法律框架。
与 Agent 焦虑的关联
当 Claude Code 作为 Agent 自主运行时，安全层级的重要性被放大——一旦模型层的价值观不够稳健，Agent 的自主行动可能带来不可预期的后果。
59:35 Anxiety when AI agents aren't working ▼

Boris 聊到了一个意外但真实的心理现象：当 Claude Code 作为 Agent 在后台运行任务时，如果很久没有输出，他会感到焦虑——就像委托了一件事给一个沉默的同事。他认为这是 AI Agent UX 需要解决的重要问题：如何让用户在等待异步任务时感到安心？

01:03:21 Advice for building AI products ▼

Boris 给 AI 产品创业者的建议：

1
找到一个真实存在的潜在需求，而不是想象中的用户痛点
2
尽量垂直整合——靠近模型能力，避免"中间层"的脆弱性
3
用 AI 工具构建 AI 产品——这不只是效率问题，更是产品理解问题
4
优先招聘能深度使用 AI 的人，而不只是写 AI 代码的人

01:14:02 Lightning round and final thoughts ▼

闪电问答环节。Boris 的几个快问快答摘录：

Q

PRO TIPS
🛠️ Claude Code 实用技巧（Boris 亲授）

📁 给 Claude 完整上下文

不要只描述你想要的功能，要让 Claude Code 先阅读和理解整个代码库（或相关模块）。使用 /read 命令让 Claude 先"学习"你的代码风格。

💬 用自然语言描述意图，不要描述实现

与其说"修改第 42 行的 for 循环"，不如说"让用户列表的加载更快"。Claude Code 在理解意图时表现最好，让它自己决定如何实现。

🔄 迭代式工作流

不要试图一次性让 Claude Code 完成所有事情。把大任务拆成小步骤，每步验证后再继续。这样既降低出错风险，也让你保持对代码的理解和控制。

🧪 让 Claude Code 写测试

总是让 Claude Code 同时生成测试用例。这不仅能提高代码质量，更能帮助你理解 Claude 实现的逻辑——测试是最好的文档。

📝 用 CLAUDE.md 文件定制行为

在项目根目录创建 CLAUDE.md 文件，写入项目约定、技术栈说明、代码风格要求。Claude Code 每次启动都会读取它，确保行为一致。

🤖 让 Claude Code 自我调试

遇到 bug 时，直接把错误信息粘贴给 Claude Code，让它自己定位和修复。大多数情况下，它比你更快找到根本原因——尤其是当你给它运行测试的权限时。

KEY TAKEAWAYS
🎯 核心要点总结

本期最值得记住的 8 件事

1
编码已被"解决"——将想法转化为代码不再是瓶颈。瓶颈已转移到"构建什么"和"为什么构建"，判断力比执行力更有价值。
2
4% 的 GitHub 提交——Claude Code 已生成全球 4% 的公开代码提交，日活用户上月翻倍，增速持续加速。
3
潜在需求是最强产品信号——当用户以非预期方式使用你的产品，这就是下一个产品机会的指针。Cowork 就是这样诞生的。
4
给工程师无限 Token——token 成本远低于工程师时间成本，消除 AI 使用限制是最好的产品研发投资。
5
用产品本身构建产品——Cowork 和 Claude Code 自身都由 Claude Code 构建。这是最快建立团队产品理解和信任的方式。
6
Cowork：10 天从零上线——AI 正在把产品开发周期从月缩短到天。这对竞争格局意味着巨大变化。
7
印刷机类比——AI 不会消灭工程师，而是让更多人能构建软件，并催生全新职业。"软件工程师"可能消失，但"构建者"的需求将爆炸式增长。
8
三层 AI 安全体系——模型层（训练期价值观）+ 产品层（使用政策）+ 社会层（外部规范），三者缺一不可。
